Description: 

After assigning an ASSUME type to some columnfamily CF in keyspace K, if a SELECT is
subsequently done on CF while the session is using a different keyspace, the ASSUME does
not take effect:
<div class="preformatted panel" style="border-width: 1px;"><div
class="preformattedContent panelContent"><pre>cqlsh&gt; USE ks;cqlsh:ks&gt; CREATE
COLUMNFAMILY cf (key int PRIMARY KEY, col int);cqlsh:ks&gt; INSERT INTO cf (key, col)
VALUES (99, 1633837924);cqlsh:ks&gt; ASSUME cf(col) VALUES ARE ascii;cqlsh:ks&gt; SELECT *
FROM cf; KEY |  col |  99 | abcd |cqlsh:ks&gt; USE system;cqlsh:system&gt; SELECT * FROM
ks.cf; KEY |        col |  99 | 1633837924 |</pre></div></div>
the output from both
{{SELECT}}s there should be the same.
