Description: 

from cassandra-cli, say type "help assume"

you will find:<br/>  Supported values
are:
<ul class="alternate"
type="square">	<li>AsciiType</li>	<li>BytesType</li>	<li>CounterColumnType (distributed
counter column)</li>	<li>IntegerType (a generic variable-length integer
type)</li>	<li>LexicalUUIDType</li>	<li>LongType</li>	<li>UTF8Type</li></ul>
ok
now:<br/><span class="error">&#91;default@cfs&#93;</span> assume inode comparator as
UTF8Type;   <br/>Type 'UTF8Type' was not found. Available: bytes, integer, long,
lexicaluuid, timeuuid, utf8, ascii.

so looks like the "supported type list should be
update by taking away the "Type" post-fix..

however, on the other hand, you can't really
use it:

<span class="error">&#91;default@cfs&#93;</span> update column family inode;     
                   <br/>Unable to find abstract-type class
'org.apache.cassandra.db.marshal.utf8'

looks like from the update, you still need the
"Type" (case insensitive?)
