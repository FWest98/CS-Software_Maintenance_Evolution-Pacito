Description: 

Additional option of 'nodetool ring' command is not documented. 

Old variant:
<div
class="code panel" style="border-width: 1px;"><div class="codeContent panelContent"><pre
class="code-java">  ring                   - Print information about the token
ring</pre></div></div>
New variant
<div class="code panel" style="border-width: 1px;"><div
class="codeContent panelContent"><pre class="code-java">  ring [keyspace]        - Print
information about the token ring <span class="code-keyword">for</span> a given keyspace
(<span class="code-keyword">for</span> all keyspaces <span class="code-keyword">if</span>
it is not specified)</pre></div></div>
Patch has been attached.

Some history (I think it
will be useful for someone who will have the same problem): <br/>I have 4-nodes cluster.
One of nodes is located in other DC (DC2). I've met the problem that 'nodetool ring' shows
that node in DC2 'Owns 0.00%'. I've found following questions:<br/><a
href="http://www.datastax.com/support-forums/topic/owns-in-nodetool" class="external-link"
rel="nofollow">http://www.datastax.com/support-forums/topic/owns-in-nodetool</a><br/><a
href="http://mail-archives.apache.org/mod_mbox/cassandra-user/201202.mbox/%3C323DBC99-8519-4E86-B69B-6520D53FA5CF@humbaba.net%3E"
class="external-link"
rel="nofollow">http://mail-archives.apache.org/mod_mbox/cassandra-user/201202.mbox/%3C323DBC99-8519-4E86-B69B-6520D53FA5CF@humbaba.net%3E</a><br/>and
task with fix:<br/><a href="https://issues.apache.org/jira/browse/CASSANDRA-3412"
class="external-link"
rel="nofollow">https://issues.apache.org/jira/browse/CASSANDRA-3412</a>

So you should use
'nodetool ring keyspace' for getting right information.
